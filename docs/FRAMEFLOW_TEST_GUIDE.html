01 · Test Intent

这套测试最终要证明什么？

验收重点不是“页面元素都点过”，而是用可复现证据证明核心业务闭环、可靠性、解释性和真实能力边界。

自动化 手工浏览器 条件能力 隔离环境
01服务真的可用API、数据库、Worker 与素材种子都已就绪，不是静态前端。
02输入不是写死样例使用全新文本，创建真实 Project 与持久 Job。
03任务真的异步202 返回后由 Worker 经历多个可观察阶段与事件。
04候选可解释每段至少 3 个唯一候选，并有分项得分、命中词和中文理由。
05人工结果不会丢编辑、排序、手选素材在 Ctrl+R 或服务重启后仍存在。
06失败能定位恢复错误码、阶段、request_id、重试语义和事件历史可追踪。
07Provider 如实记录规则、LLM、降级和 Token 以运行记录为准，不包装成不存在的 AI。
08部署可交付构建、Docker、持久卷、HTTPS 与备份恢复都有明确验收路径。

03 · Prepare & Run

需要什么，以及怎么启动

文本主流程不依赖任何外部 AI。真实音视频转写属于条件能力，需要额外安装 ASR 依赖或配置外部 Provider。

必须准备

Py
Python 3.11+正式 CI 基线为 Python 3.12；当前机器为 3.13.1。
JS
Node.js 22+ 与 npmCI 使用 Node 22；当前机器版本更高，发布前仍要以 CI 为准。
PS
PowerShell + Chrome / Edge用于启动、验收脚本、桌面与移动视口检查。
DB
API、Worker、SQLite 数据目录API 与 Worker 必须指向同一个 FRAMEFLOW_DATA_DIR。
HD
至少 2 GB 可用磁盘安装依赖、构建产物、上传媒体与测试证据需要空间。

可选准备

ASR
faster-whisper 或外部 ASR仅用于真实音视频转写；缺少时应测试明确失败，不生成假字幕。
LLM
OpenAI-compatible / DeepSeek仅增强分段、主题与关键词；模型失败可规则降级。
DK
Docker Desktop / Linux Docker用于容器、持久卷、健康检查与发布验收。
NET
域名、HTTPS 与访问控制公网演示必须限制访问，尤其是上传与故障注入接口。
!
开始正式测试前，先完整重启当前服务

仓库当前有未提交修改，运行中的本地进程也可能早于最新代码。先记录最终 commit / 工作树状态，再重启 API、Worker 和前端，避免用旧进程画面证明新代码。

适合：本机演示、第一次启动。脚本会检查环境、安装运行依赖、启动 API + Worker + Vite。

cd C:\path\to\frameflow
.\start-frameflow.cmd
△
一键启动不会自动安装测试和本地 ASR 依赖

要跑 pytest，需要安装 requirements-dev.txt；要做本地转写，需要另装 requirements-asr-local.txt。

先安装开发依赖：

cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt -r requirements-dev.txt

终端 A · Worker：

cd backend
.\.venv\Scripts\Activate.ps1
python -m app.worker

终端 B · API：

cd backend
.\.venv\Scripts\Activate.ps1
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000

终端 C · 前端：

cd frontend
npm ci
npm run dev
Copy-Item deploy\.env.example deploy\.env
docker compose --env-file deploy\.env config --quiet
docker compose --env-file deploy\.env build
docker compose --env-file deploy\.env up -d
docker compose --env-file deploy\.env ps

容器镜像包含 ffmpeg，适合验证媒体处理运行环境。重启容器时不要删除 volume，随后打开同一 Project ID 验证持久化。

backend\.venv\Scripts\python.exe -m pip install -r backend\requirements-asr-local.txt

首次使用 faster-whisper 会下载模型，需要额外时间、网络与磁盘。当前 Windows PATH 未确认存在 ffmpeg CLI；Docker 镜像已安装 ffmpeg。没有完成这些准备时，音视频任务进入明确的 ASR 配置/依赖失败才是正确结果。

05 · Core Flow

15 分钟核心闭环：一步一步怎么做

每一步都包含“怎么做、为什么做、通过时应该看到什么、它能展示什么能力”。

确认 live / ready 和首页

打开两个健康端点和前端项目台。ready 必须明确反映数据库、Worker 和素材状态。

用意排除“只有静态页面”的假成功。
通过标准live=200，ready=200，首页无网络错误。
展示能力服务端架构和运行就绪检查。

创建一个全新文本项目

进入“新建项目”，输入新标题，粘贴 TD-01 或新的 100 字以上文本，点击创建。提交时按钮应禁用。

用意证明系统能处理任意新输入。
通过标准进入处理页，只创建一个 Project/Job。
展示能力真实表单、202 创建和幂等防重。

观察真实异步阶段与事件

停留在处理页，至少看到 validating、segmenting、keywording、matching、persisting 中的多个阶段，进度应单调增长。

用意证明耗时工作不是浏览器里假动画。
通过标准事件来自服务端，成功后可进入工作台。
展示能力独立 Worker、持久任务和可观测进度。

检查原文、分段、主题和关键词

进入工作台，切换“内容片段 / 原始字幕”，依次打开不同片段，确认原文完整，每段非空且有主题、关键词。

用意证明分段不是单条写死结果。
通过标准不同语义对应不同片段与关键词。
展示能力中文分段、关键词抽取和输入留痕。

检查每段至少 3 个唯一可解释候选

对每个片段查看候选，展开“匹配依据”，检查总分、TF-IDF、关键词、主题得分、命中词和中文解释。

用意证明匹配不是黑箱。
通过标准每段 ≥3 个不同素材，理由非空。
展示能力透明混排与低相关补位诚实标记。

修改字幕、主题、关键词并调整顺序

编辑当前片段，观察“等待保存 → 保存中 → 已保存”，再用拖动或上下箭头改变片段顺序。

用意证明 AI 只是初稿，人能修正。
通过标准保存反馈清楚，排序完成无重复位置。
展示能力自动保存、乐观锁和事务排序。

全库搜索并人工替换素材

搜索“自然”“健康”或“数据”，选择一个不是当前选择的素材；随后点击重新匹配。

用意证明用户是最终决策者。
通过标准显示 manual/人工选择，重新匹配不覆盖手选。
展示能力素材检索、人工选择优先级。

Ctrl+R 强制刷新，证明持久化

刷新同一项目，重新找到刚才片段，核对文本、主题、关键词、顺序和素材选择。

用意这是整个项目最关键的真实性证据。
通过标准全部修改仍在，来自 SQLite 重读。
展示能力完整的持久化业务闭环。

08 · Reliability Drills

故障、冲突与恢复怎么测

这些测试的用意，是展示系统不仅能走成功路径，还能如实失败、定位原因、安全恢复，并避免重复结果。

只在隔离数据或专用演示项目执行
一次性故障

AI 降级 ai_degrade

  1. 演示实验室选择“下一任务 AI 降级”。
  2. 新建 TD-02 文本项目。
  3. 等待项目成功。
  4. 运行记录检查 degraded=true 与真实 Provider。

展示：外部增强失败时，确定性规则仍能完成业务闭环。

一次性故障

任务失败 job_fail → retry

  1. 设置下一任务失败并创建项目。
  2. 检查错误码、阶段、原因、attempt、request_id。
  3. 点击重试，确认没有新建项目。
  4. 检查 attempt+1、历史事件保留、最终成功。

展示：持久失败状态、可恢复性和正确的重试语义。

双页面

乐观锁冲突 409

  1. 在两个浏览器页打开同一项目和片段。
  2. A 页编辑并保存。
  3. B 页基于旧 version 保存。
  4. 检查 B 页保留草稿并显示冲突提示。

展示：不会让旧页面静默覆盖新数据。

进程操作

Worker 租约恢复

  1. 临时目录设置 5 秒租约与较慢阶段。
  2. 任务 running 时终止 Worker。
  3. 等待租约过期，重启 Worker。
  4. 确认任务恢复且 Segment/Recommendation 不重复。

展示：durable queue、心跳、租约接管和执行代次栅栏。

服务重启

API / 容器重启持久化

  1. 记录项目 ID、编辑内容和人工选择。
  2. 重启 API 或容器，不删除 data/volume。
  3. 重新打开同一 URL。
  4. 检查项目、事件、编辑、顺序、素材仍在。

展示：数据不依赖浏览器内存或进程内存。

配置恢复

ASR 修复后原任务重试

  1. 在无 ASR 依赖/Key 时上传媒体。
  2. 确认明确配置错误且输入已保存。
  3. 补齐依赖或 Provider 配置并重启。
  4. 重试同一任务，无需重新上传。

展示：输入持久化、错误分类和可操作恢复建议。

13 · Honest Boundaries

这些能力已经有实现，但不能越过真实边界

清晰披露边界反而会增强可信度。运行时 Provider 和能力状态应以实际配置、任务事件与运行记录为准。

!
音视频上传 ≠ 默认 ASR 已成功上传与异步任务已实现；转写需要 faster-whisper 或真实外部 ASR。
!
默认 TF-IDF ≠ 神经 Embedding默认使用字符 n-gram；只有运行记录显示本地 BGE 或远程 embeddings 时，才能宣称启用了神经向量。
!
自动标签不是视觉模型识图上传素材留空时可由 LLM 或规则分析名称/文本元数据并记录来源；当前不识别真实画面内容。
!
组合预览不是精确视频剪辑时间轴已支持片段时间线和 MP4 预览，但没有帧级裁切、复杂转场、多音轨或专业导出参数。
!
有界 Worker 池不代表水平扩容SQLite WAL + 默认双 Worker 适合单机 Demo 和低并发并行处理。
!
单管理员登录 ≠ 多租户权限已有应用会话、CSRF 与整站 Basic Auth，但没有团队用户、项目归属、RBAC 或租户隔离。
!
IMAGE_API_* 是预留配置当前产品没有业务内图像生成，LLM 也只增强分段、主题和关键词。
!
素材删除有保护边界只能删除未被项目引用的用户上传素材；种子、已引用和最低保底素材会拒绝删除。项目重命名与审计管理页面仍未完成。
!
公网 Demo 不是生产安全系统已有单机读写限流，但没有分布式配额、恶意文件扫描、RBAC 与多租户隔离。
!
文档与运行实例必须重新核对发布前校对 KNOWN_ISSUES、AI_USAGE、配置变量和实际代码，避免陈旧描述。
