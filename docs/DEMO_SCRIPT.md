# FrameFlow AI 4 分 30 秒演示脚本

> 交付状态：**待补充（脚本已准备）**。本文只是正式 3～5 分钟实机录屏的讲解与镜头计划，不是演示视频本身，也不能替代题目要求的视频文件或可访问链接。

## 1. 演示目标

在 4 分 30 秒内用一段新文本证明以下事实：

1. 项目创建和耗时处理是真实、持久化的后端流程。
2. 字幕会被分段并生成关键词/主题。
3. 每段至少有 3 个不同候选，且显示命中词和匹配理由。
4. 人工编辑、排序和替换会保存，强制刷新不丢失。
5. 应用登录、素材上传/删除和危险操作保护真实可用。
6. 现有候选不足时，可按字幕文生图；确认入库后再异步生成视觉标签并用于当前片段。
7. 时间线与 MP4 组合预览通过持久化异步任务生成。
8. 系统可重现任务失败，显示原因并安全重试。
9. 运行记录如实标识实际 Provider，不伪造外部 AI、Embedding、视觉标签或 ASR 结果。

## 2. 录制前 10 分钟检查

- 使用即将提交的 commit，记下 commit SHA。
- 在同一环境启动 API 与 Worker，确认 `/health/live` 和 `/health/ready` 为成功。
- 运行 `scripts/acceptance.ps1` 或 `scripts/acceptance.sh`；如未通过，不开始录制。
- 打开公网演示地址，不用 localhost 冒充公网环境。
- 用无痕窗口验证应用登录页、测试账号和退出后重新登录；不要在录屏中展示密码管理器或服务器 `.env`。
- 只有在受控演示窗口临时启用 `DEMO_MODE` 时才展示故障实验室；开始前将模式清为 `none`，结束后立即关闭。正式长期公网环境保持 `DEMO_MODE=false`。
- 确认 30 个内置种子素材（24 张图片 + 6 个视频）均可加载；公网实时素材总数以页面为准，且图片无破图、视频有 poster。
- 确认本地 `faster-whisper small/int8` 模型已缓存、核心 Worker 按公网配置单并发运行且任务队列为空；另备一段已真实复验的短音频作为 ASR 证据，不在 4:30 主流程中等待长音频。
- 准备一份未被项目引用的测试图片或短视频，用于素材上传与删除；不要尝试删除种子素材作为成功案例。
- 确认 Image Worker 在线，并准备一条真实成功、仍可确认入库的文生图任务，以及一份已经完成视觉标签的生成素材。现场展示已有真实结果及其耗时，不把预先完成的调用说成即时生成。
- 确认图像生成和视觉标签的运行记录可看到实际 Provider、Model、耗时与降级状态；任何截图、录屏和浏览器网络面板都不能出现 `IMAGE_API_KEY`、`VISION_API_KEY` 或请求头。
- 预先复制 `demo/sample-transcript.txt` 和 `demo/failure-retry-transcript.txt`，但不预先创建主演示项目。
- 建议浏览器 1280×720 或 1440×900、90%～100% 缩放，隐藏书签栏和无关通知。

## 3. 主演示文本

```text
清晨，我骑着自行车穿过城市，准时来到办公室。团队用数据看板梳理项目进度，把复杂任务拆成清晰步骤。午后，我们到公园散步，讨论怎样用绿色科技让工作更高效，也让生活更健康。
```

该文本故意覆盖城市/交通、团队/数据/办公、自然/健康/绿色科技三组语义，便于在短时间内直观检查排序是否随内容变化。

## 4. 4:30 逐镜头时间表

### 镜头 1｜00:00–00:18｜登录与业务闭环

**画面**：从登录页开始，使用提交给面试官的临时测试账号登录，进入“项目”页。画面同时可见 FrameFlow 品牌、深色侧栏中的“项目 / 素材库 / 运行记录 / 演示工具”和橙色“新建项目”。

**口播**：

> FrameFlow AI 是一个字幕到素材的轻量工作台。应用使用服务端会话和 CSRF 保护写操作；登录后进入真实业务闭环：输入、异步处理、可解释匹配、人工修正和持久化。

**证据点**：真实登录页、会话保护、产品范围清晰，首屏有真实服务端数据。

### 镜头 2｜00:18–00:45｜用新文本创建项目

**画面**：点击“新建项目”。快速切换一次“上传音视频” Tab，展示真实文件选择入口与 ASR 配置提示，然后回到“粘贴文本”。填写名称“演示－绿色高效的一天”，粘贴主文本，点击创建。

**口播**：

> 系统同时接受音视频和文本。公网音视频由服务器上的 faster-whisper small/int8 真实转写，避免跨境回源；这段 4 分半演示为了集中展示后续编辑闭环，使用同样受支持的文本入口。这里提交一段带城市、数据和绿色科技的新文本。

**证据点**：上传 UI 真实存在；ASR 结论有独立运行记录支撑；新文本不是程序内写死字符串。

### 镜头 3｜00:45–01:05｜真实异步进度

**画面**：创建返回后自动进处理页。停留，让录屏看到至少 `validating/segmenting/keywording/matching/persisting` 中的多个真实事件。

**口播**：

> API 只在事务里创建项目和持久化任务，然后返回 202。独立 Worker 从 SQLite 原子领取任务。这些阶段、进度和事件都来自服务端，刷新或进程重启后不会因前端内存丢失。

**证据点**：HTTP 202 语义、独立 Worker、服务端事件。

### 镜头 4｜01:05–01:45｜分段与可解释候选

**画面**：进入工作台。从左栏指向原始字幕与 3 个语义片段。切换“城市骑行”、“数据看板”和“绿色科技”片段，每次在右栏滑过前 3 个候选。展开其中一张的分项或解释。

**口播**：

> 文本被通用标点和长度规则切分，每段提取主题和关键词。右侧每段都有至少三个不同候选。排序不是只给一个神秘分数：55% 是语义相似度，30% 是关键词重合，15% 是标签和主题重合；语义通道实际使用字符 n-gram 还是 Embedding，以运行记录为准。这里可以看到命中词和中文理由；如果是补位候选，也会明确标记。

**证据点**：原文、通用分段、关键词、三候选、分项与理由。

### 镜头 5｜01:45–02:20｜编辑、排序与素材替换

**画面**：选中第二片段，在文本末补入“并持续检查风险。”，增加关键词“风险”并保存。展示“保存中→已自动保存”，再拖动调整顺序。随后选中“绿色科技”片段，全库搜索“自然”或“健康”，采用一张非当前素材。

**口播**：

> 自动分析是初稿，人才是最终决策者。文本修改带版本号，顺序通过整体事务保存；全库搜索可以快速替换，手工选择会标为 manual，不会被重新匹配静默覆盖。

**证据点**：可编辑、保存反馈、乐观锁、排序事务、全库替换和 manual 优先。

### 镜头 6｜02:20–02:55｜刷新、时间线与 MP4 预览

**画面**：按 Ctrl+R 刷新，指向刚才保存的文本、顺序和素材。打开素材时间线，点击一个片段定位字幕；创建组合预览，展示预览 Job 进度，完成后播放 MP4。录制前可准备同一输入指纹的成功结果，现场点击时仍要说明是否复用了幂等结果。

**口播**：

> 强制刷新后，文本、顺序和手工选择都从 SQLite 重新加载。时间线按字幕时间戳或确定性估算时长组织素材，预览通过独立持久化任务交给 ffmpeg 生成；相同输入指纹不会重复制造活动任务。

**证据点**：刷新持久化、片段时间线、预览进度、幂等与真实 MP4 播放。

### 镜头 7｜02:55–03:20｜文生图与 Gemini 视觉标签

**画面**：回到工作台候选区，点击“按本段字幕生成一张新画面”。展示一条录制前已经真实完成的任务：提示词、16:9 比例、真实模型、耗时和“图片已生成”状态；明确说它不是现场瞬时生成。点击“使用并加入素材库”，随后打开准备好的生成素材详情，指向“画面识别完成”、Gemini 根据真实画面生成的标签/关键词和重新生成标签入口。

**口播**：

> 当候选都不合适时，FrameFlow 不会静默发起付费调用，必须由用户明确提交。生成结果先留在私有草稿区，确认后才原子加入素材库并用于当前片段，再排队执行视觉标签。这里的标签确实来自本次 Gemini 画面识别；如果视觉服务不可用，界面和运行记录会如实标成文本或规则降级。

**证据点**：真实文生图任务与耗时、用户确认入库、`generated` 选择、Gemini 视觉标签、降级来源不冒充视觉结果。

### 镜头 8｜03:20–03:35｜素材治理与 AI 运行追溯

**画面**：在素材详情快速指向编辑与删除入口，并说明当前片段正在引用的素材会拒绝删除。打开“AI 运行记录”，让 `文生图素材生成`、`素材智能标注`、字幕语义增强和预览渲染的实际 Provider、Model 与耗时同屏。

**口播**：

> 素材的新增、标签和选择都可追溯。只有未被引用的用户素材可以删除；种子素材和项目正在使用的素材会被保护。运行记录显示什么 Provider，我就按什么 Provider 解释。

**证据点**：素材治理、实际 Provider/Model、文生图与视觉标签调用留痕。

### 镜头 9｜03:35–04:15｜可重复失败与重试

**画面**：点击侧栏“演示工具”，进入页面内的“演示实验室”，指向“这是故障演练”警告。在“模拟任务失败”卡片点击“设置一次”，立即新建文本项目“故障演练”，粘贴 `failure-retry-transcript.txt`（若时间紧，只粘贴前两句）。看到 failed 状态、错误原因和重试按钮后点击重试，等待至少进入 running；如处理速度足够，停到 succeeded。

**口播**：

> 这是明确标注的一次性故障演练，用来稳定验收失败路径。任务失败后保留已上传的输入、错误码、失败阶段和事件历史。点击重试会增加尝试次数并重新入队，不是又创建一个项目。故障只消费一次，所以重试可正常完成。

**证据点**：故障明确标注、持久化 failed、原因、事件历史、重试语义。

### 镜头 10｜04:15–04:30｜取舍和结束

**画面**：回到主项目工作台，定格在字幕、预览和三候选同时可见的画面。

**口播**：

> 这个 Demo 的边界也是明确的：它是单实例 SQLite、单并发核心 Worker 与独立单并发 Image Worker 的公网配置，不是水平扩容集群；专用服务器的一次 71 秒热机样本中，本地 ASR 约 20.5 秒、Gemini 语义增强约 3.1 秒、完整流程约 26 秒，这些是实测样本而不是 SLA；组合预览不是帧级剪辑器，也没有多音轨、复杂转场或专业导出参数。但登录、异步闭环、可解释匹配、人工持久化、文生图入库、视觉标签和 MP4 预览都可以独立复现。

**证据点**：主动披露边界，不夸大产品能力。

## 5. 时间超支处理

- 若主任务处理超过 22 秒，口播继续解释当前事件，不剪掉等待并伪装瞬时完成。可缩短第 8 镜的素材治理停留，但保留 20～30 秒文生图/视觉标签镜头。
- 音视频 ASR 不塞进 4:30 主流程；需要展示时单独录制短片，从上传开始保留完整进度、运行记录和最终字幕，不剪掉等待伪装即时完成。
- 文生图真实调用通常需要数十秒且可能计费，4:30 主视频使用录制前真实完成的持久任务，并展示其模型、创建时间和实际耗时；不能剪掉等待后声称“现场几秒生成”。如视觉标签尚在排队，只展示真实状态，另用已完成素材证明最终结果。
- 若故障重试未在 04:15 前完成，只要已显示 queued/running 且尝试次数增加，即可进入结束语；录制后需单独复验它最终 succeeded。
- 若公网环境当场出现非注入失败，保留错误画面并说明 request_id，再根据可重试性操作。不用前端假数据跳过。

## 6. 演示中禁止的说法

- “这里调用了大模型”，如果运行记录实际为 `rules`。
- “这是语义 Embedding”，如果实际为字符 n-gram TF-IDF。
- “任意音视频都能固定在 20.5 秒内转写”；20.5 秒只是一次 71 秒热机样本的 ASR 阶段记录。
- “这些标签来自 Gemini 视觉识别”，如果素材详情或运行记录实际显示文本 LLM、规则降级或尚未执行；只有真实 `vision` 来源才能这样说明。
- “重新生成不会再次计费”；每次明确重新调用都可能产生新的供应商费用，结果未知状态必须先核对再由用户确认。
- “这是精确视频剪辑时间线”，当前是按字幕片段组织的组合预览。
- “已支持高并发/水平扩容”，当前公网是单并发核心 Worker 与独立单并发 Image Worker 的单机有界执行，而不是高吞吐集群。
- “这是真实 AI 宕机”，对一次性故障注入应说“可重复故障演练”。

## 7. 录制后校验

- 时长在 4:15～4:45 之间，目标 4:30。
- 登录、项目创建、处理事件、三候选、人工替换、强制刷新、时间线/MP4、文生图确认入库、Gemini 视觉标签、故障和重试画面都在。
- 全程无 API Key、`.env`、个人账号、私有仓库 URL 或本地绝对路径。
- 画面可看清错误原因和匹配理由，不只靠口播。
- 视频描述附上公网地址、源码地址、测试账号（如有）和录制对应 commit。
