# FrameFlow AI 评分对照矩阵

## 使用说明

本文是交付自查表，不是对面试得分的承诺。一项能力只在其对应自动化或手工证据已通过时才能标记为完成。

状态含义：

- **已实现**：代码路径存在，交付前仍需按 `TEST_PLAN.md` 复验。
- **已实现（需配置）**：实现和自动化证据存在，但是否实际启用取决于部署环境；验收时必须以运行记录为准。
- **部分**：稳定入口或适配器存在，但有如实披露的外部依赖/能力边界。
- **未完成 / 未实现**：交付物尚未提供，或功能属于本次主动取舍；不以脚本、截图或模拟结果冒充完成。

### 题目外业务扩展

文生图闭环已经实现：用户显式提交自然语言提示词后，独立 Image Worker 生成私有草稿；确认后才原子加入素材库、排队三级标签，并可应用到当前字幕片段。该能力使用独立 Provider/密钥、单并发、每日与待处理配额和结果未知防重复计费屏障。它可以作为闭环扩展展示，但不能挤占题目规定的输入、字幕、匹配、人工替换和刷新持久化主流程。

## 五、得分风险与冲刺顺序

当前音视频路径已用真实本地 ASR 验证；最高风险转为把单次耗时误说成稳定 SLA，或忽略本地推理导致的排队。专用公网机保持单 Worker并给容器分配 3.5 CPU / 4 GB，一次 71 秒热机音频的 ASR 阶段约 20.5 秒、完整流程约 26 秒；演示应展示真实进度与运行记录，不承诺所有音频固定耗时。

发布前准出顺序：

1. 公网环境完整跑通文本闭环、替换和刷新持久化。
2. 跑通 `job_fail → failed → retry → succeeded` 与 `ai_degrade → succeeded + degraded`。
3. 登录后完成素材上传、单画面视觉标签、人工修订和安全删除；核对实际 `source=vision/text_llm/rules`，并验证种子/已引用素材拒绝删除。
4. 生成并播放一次真实 MP4 组合预览，明确它不含原音轨、不是专业剪辑成片；保存任务记录与输出证据。
5. 用新提示词完成一次文生图草稿→确认入库→后台标签→字幕片段应用，并说明提示词隐私、可能费用和结果未知重试边界。
6. 保存 CI、本地最终门禁、截图或视频时间点作为证据，并提供真实 3～5 分钟演示视频链接。
7. 以短音频复验本地 `faster-whisper small/int8`，确认运行记录不是规则或假数据；另保留无 Provider 的明确失败测试。
8. 在真实域名完成 HTTPS、登录、持久化重启与 acceptance，不用本地结果替代公网证据。

2026-07-15 文生图与时间线节奏融合基线的本地门禁为：后端 `222 passed, 1 deselected`，文生图专项 `54 passed`，前端 lint、TypeScript 与生产构建通过，Chromium Playwright `48 passed`，启动器契约 `7 passed`。这些是对应代码基线的本地结果；真实 Gemini、Vision、Image、ASR 与公网行为必须以同一发布提交的运行记录、smoke 和人工 E2E 为准。

演示中不得把字符 n-gram TF-IDF 称为当前生产神经 Embedding，不得把单张图片或视频 poster 的标签称为整段视频理解，也不得把无原音轨的组合预览称为完成剪辑的成片。文生图超时或断流后的再次调用可能二次计费，必须由用户确认，不能包装成无成本自动重试。

在全部必须流程、故障流程、公网部署和文档证据都通过后，本方案才具备冲刻 95 分的证据基础；实际分数仍由面试验收决定。
