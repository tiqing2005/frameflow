# FrameFlow AI 评分对照矩阵

## 使用说明

本文是交付自查表，不是对面试得分的承诺。一项能力只在其对应自动化或手工证据已通过时才能标记为完成。

状态含义：

- **已实现**：代码路径存在，交付前仍需按 `TEST_PLAN.md` 复验。
- **部分**：稳定入口或适配器存在，但有如实披露的外部依赖/能力边界。
- **未实现**：本次主动取舍，不在演示中伪造结果。

## 四、得分风险与冲刺顺序

最高风险是把“音视频可上传”误说成“默认已有真实 ASR”。题目允许无语音服务时保留文本稳定入口，因此正确策略是如实披露，而非用假转写换表面完成度。

发布前准出顺序：

1. 公网环境完整跑通文本闭环、替换和刷新持久化。
2. 跑通 `job_fail → failed → retry → succeeded` 与 `ai_degrade → succeeded + degraded`。
3. 登录后完成素材上传、自动元数据建议、编辑与安全删除；验证种子/已引用素材拒绝删除。
4. 生成并播放一次真实 MP4 组合预览，保存任务记录与输出证据。
5. 保存 CI、本地最终门禁、截图或视频时间点作为证据。
6. 若部署环境有稳定 ASR Key，增加短音频实测；没有则保留明确失败与文本主流程。
7. 在真实域名完成 HTTPS、登录、持久化重启与 acceptance，不用本地结果替代公网证据。
