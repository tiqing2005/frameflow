# FrameFlow AI 产品需求文档（PRD）

| 字段 | 内容 |
| --- | --- |
| 产品名 | FrameFlow AI |
| 版本 | Demo 1.0 |
| 目标交付 | 招聘实战题二：AI 字幕分析与素材匹配平台 |
| 主语言 | 简体中文 |
| 产品形态 | 可独立操作的公网 Web 工作台 |
| 优先级 | P0 真实闭环 > P0 稳定性 > P1 可解释/追溯与素材生成加分项 > P2 专业剪辑能力 |

## 1. 问题背景

短视频制作者拿到一段口播文稿后，需要先理解内容、切成可编辑片段，再为每段查找可用的图片或短视频素材。纯手工流程不仅耗时，而且难以回答“为什么这个素材适合这句话”。

FrameFlow AI 不试图成为完整剪辑器，而是聚焦最可验收的中间环节：把文本/媒体输入变成可编辑字幕片段，给出可解释的素材候选，允许人工修正，并保存最终对应关系。

## 2. 产品目标

### 2.1 用户目标

- 在一个界面中完成“输入→分段→候选→人工调整→保存”。
- 了解长时处理当前处于哪个阶段，失败时知道原因和下一步。
- 每个字幕片段能比较至少 3 个不同素材，并看到命中词和匹配理由。
- 替换或重排后无需人工改数据库，刷新页面不丢失结果。
- 素材库没有合适画面时，可用自然语言生成图片，审核后再入库或应用到当前字幕片段。

### 2.2 交付目标

- 公网环境可由面试官独立操作。
- 对新的字幕文本有效，不依赖演示样例写死。
- 任务、结果、人工选择和审计信息持久化。
- 外部 AI 不可用时仍有确定性规则结果，或给出真实且可重试的失败。
- 用文档、测试和运行记录证明能力，不伪造 ASR/LLM 调用。

## 3. 非目标

- 不做帧级视频剪辑、多轨时间线、转场和调色。
- 已提供可播放的 MP4 组合预览，但不把它定义为专业剪辑导出，不承诺码率、分辨率、转场、音轨和导出模板等专业参数。
- 不承诺多租户、RBAC、高并发或跨机水平扩容。
- 不把 TF-IDF 宣称为神经语义 Embedding，不把故障注入宣称为外部 Provider 真实故障。
- 默认无 ASR Provider 时不生成假字幕。

## 4. 目标用户与使用场景

| 角色 | 核心需求 | 成功信号 |
| --- | --- | --- |
| 内容编辑 | 快速把口播文稿分段并选素材 | 在数分钟内完成一个可保存的匹配方案 |
| 素材管理者 | 上传素材并维护可搜索元数据 | 素材能被搜索且进入推荐 |
| 演示/验收者 | 看到真实状态、失败、重试和持久化 | 不借助开发者手工改数据即可完成验收 |

## 5. 主用户旅程

### 5.1 文本主流程（稳定演示入口）

1. 用户进入“新建项目”，填写项目名称并粘贴一段新文本。
2. 提交后 API 在事务内创建项目与持久化任务，返回 202。
3. 页面显示真实阶段、进度和事件，不用前端假计时伪装处理。
4. Worker 完成分段、关键词提取、混合检索与事务化保存。
5. 用户进入三栏工作台，查看原文、片段、当前预览和候选。
6. 用户编辑一段文本，重新排序，搜索素材并替换当前选择。
7. 页面明确显示保存中/已保存；强制刷新后依然可见人工结果。
8. 用户可按当前字幕顺序查看素材时间线，设置目标总时长或单片段展示时长，并可恢复自动节奏。
9. 节奏变化后旧预览明确显示为已过期；用户确认后再通过持久化异步任务重新生成、播放 MP4 组合预览。

### 5.2 上传流程

1. 用户选择音频或视频并提交。
2. 系统检查文件类型与大小，使用服务端安全文件名持久化。
3. 如配置真实 ASR，任务继续转写；否则如实失败并显示 `ASR_NOT_CONFIGURED`。
4. 失败详情告知用户该问题是否可重试，不返回伪造字幕。

### 5.3 故障演练流程

1. 用户从侧栏“演示工具”进入“演示实验室”，明确选择“下一次 AI 降级”或“下一次任务失败”。
2. `ai_degrade` 被一次性消费后，规则流水线完成任务，运行记录标记降级。
3. `job_fail` 被消费后，任务显示失败原因与重试按钮。
4. 用户点击重试，一次性故障不再继承，任务可成功完成。

### 5.4 素材理解与文生图闭环

1. 用户上传图片或短视频时可留空标签和关键词；素材先完成安全入库，随后由持久化 Worker 异步处理自动标签。
2. 已配置视觉 Provider 时，图片使用归一化画面、视频使用 poster 或抽取帧生成标签；视觉不可用时依次降级到文本 LLM 和本地规则，并在运行记录中保存实际来源。
3. 用户可从素材库独立创建图片，也可在字幕片段候选不足时发起“AI 生成图片”，填写自然语言提示词并选择画幅。
4. 图像请求进入独立的持久化 Image Worker；页面展示真实 `queued/running/succeeded/failed/canceled` 状态，刷新或进程重启后任务仍可查询。
5. 成功结果先保存在私有草稿区，不会自动成为公开素材。用户确认后，系统原子创建 Asset、排队自动标签，并可把当前 Selection 写为 `source=generated`。
6. 若生成期间字幕版本已变化，系统拒绝覆盖新字幕，但允许用户选择“仅加入素材库”；丢弃或过期草稿由受控清理流程移除。

## 6. 功能需求

### P0：必须完成

| ID | 需求 | 验收标准 |
| --- | --- | --- |
| FR-01 | 文本新建项目 | 2～100,000 字符的新文本可创建，重复点击不产生重复资源 |
| FR-02 | 音频/视频上传 | multipart 真实上传；非法类型/超限有明确反馈 |
| FR-03 | 异步任务 | 创建返回 202；持久化 queued/running/终态与多阶段事件 |
| FR-04 | 失败与重试 | 显示错误码、人类可读消息、是否可重试；重试有审计记录 |
| FR-05 | 字幕分段 | 对新文本输出一个以上非空片段，保留原始字幕 |
| FR-06 | 关键词/主题 | 每个片段显示主题或关键词；不对演示文本写死 |
| FR-07 | 片段编辑与顺序 | 文本/主题/关键词可保存；顺序无重复/遗漏 ID；版本冲突返回 409 |
| FR-08 | 素材库 | 30 个可离线加载的内置种子（24 图 + 6 视频）；可上传图片/短视频并编辑元数据 |
| FR-09 | 素材搜索 | 按名称、标签或关键词检索；无结果有空状态 |
| FR-10 | 混合推荐 | 每段至少 3 个唯一候选，保存分项得分、命中词、理由与补位标识 |
| FR-11 | 人工选择 | 用户可从候选或搜索结果替换，`source=manual` |
| FR-12 | 刷新持久化 | 项目、分段、候选、人工选择和任务事件刷新后存在 |
| FR-13 | 追溯 | 运行记录包含 Provider/Model/策略版本/输入摘要/结果摘要/降级标识 |
| FR-14 | 故障注入 | Demo 模式下可重现 AI 降级和任务失败/重试 |
| FR-15 | 时间线与组合预览 | 按字幕时间戳或安全估算时长生成素材时间线；通过持久化异步任务输出可播放 MP4，相同输入指纹可幂等复用 |
| FR-16 | 预览节奏控制 | 支持目标总时长、按字幕字数/当前比例/平均分配、单片段 1–30 秒和恢复自动；保存后旧预览失效但不自动重渲染 |

### P1：已完成的高价值加分项

| ID | 需求 | 验收标准 |
| --- | --- | --- |
| FR-17 | 自动素材标签 | 留空元数据的新素材异步生成标签/关键词；视觉、文本或规则降级来源及 Provider/Model 可追溯，已有素材可人工触发重新生成 |
| FR-18 | 文生图素材闭环 | 支持素材库和字幕片段两个入口；生成任务持久化，草稿经确认后幂等入库、排队标签，并可在版本校验通过时应用到片段 |

- 片段拖动排序，移动端使用上/下移控件作后备。
- 候选卡片一键替换，以及全库搜索替换。
- 当前片段的 16:9 暗色预览、字幕叠层和组合预览进度。
- 项目审计流和 AI/匹配运行记录页。
- 取消、幂等、租约恢复和可配置的有界 Worker 并发。
- 图片/视频画面自动标签，失败时按“视觉模型→文本 LLM→本地规则”降级并如实记录。
- 自然语言生图、私有草稿审核、素材入库、自动标签和字幕应用的一体化闭环。

### P2：明确延后

- 精确字幕时间轴。
- 帧级裁切、多轨混音、复杂转场和专业导出参数。
- 多用户权限、组织和审批。

## 7. 状态与业务规则

### 项目状态

```text
queued → processing → ready
                    ├→ failed
                    └→ canceled
failed → processing（重试）
```

### 任务状态

```text
queued → running → succeeded | failed | canceled
failed → queued（仅允许可重试任务）
```

阶段集合为 `validating | extracting | transcribing | segmenting | keywording | matching | persisting | completed`。文本任务允许跳过媒体阶段，但进度不能倒退。

### 选择规则

- 每个片段最多一个当前 Selection。
- 初始推荐可产生 `source=auto`；用户从候选或搜索结果替换时必须是 `source=manual`。
- 用户确认生成图片并应用到片段时使用 `source=generated`；仍受片段乐观锁和素材 active 状态校验。
- 重新匹配不得静默覆盖已有人工选择。
- 删除/禁用已被选择素材应受外键或业务校验保护。

### 素材标签状态

```text
idle → queued → running → succeeded | degraded
```

`succeeded` 只表示视觉 Provider 返回了通过校验的标签；使用文本 LLM 或本地规则完成时标记 `degraded`，不伪装成视觉识别成功。重新生成标签会增加 generation，并以租约和 attempt 防止旧 Worker 覆盖新结果。

### 文生图状态

```text
queued → running → succeeded | failed | canceled
failed → queued（仅在服务端判定可重试且用户确认后）
```

生成成功和“已加入素材库”是两个正交状态：`succeeded` 仅代表私有草稿可审核，只有 `asset_id` 非空才代表已幂等入库。外部响应结果不确定时不自动重复付费调用，由用户看到明确提示后决定是否重试。

## 8. 非功能需求

| ID | 类型 | 要求 |
| --- | --- | --- |
| NFR-01 | 可恢复性 | API 进程重启不丢项目/任务；过期 running 任务可被 Worker 恢复 |
| NFR-02 | 一致性 | 字幕结果与推荐换版在事务中完成；不展示半套结果 |
| NFR-03 | 幂等 | 相同作用域 + `Idempotency-Key` + 相同请求不创建新项目 |
| NFR-04 | 可解释 | 每个候选保存三个分项、命中词和中文解释 |
| NFR-05 | 秘密管理 | 任何 Provider Key 不出现在前端、数据库或公开仓库 |
| NFR-06 | 可用性 | 加载、空数据、失败、保存中/成功和重复操作有明确反馈 |
| NFR-07 | 响应式 | 桌面三栏；窄屏切换为分页/标签导航，核心操作不溢出 |
| NFR-08 | 可追溯 | 操作与任务日志有 `request_id/job_id`，可从 UI 定位服务端事件 |
| NFR-09 | 付费调用安全 | 文生图使用独立凭据、单并发 Worker、待处理/每日配额和调用次数上限；结果不确定时不静默重发 |
| NFR-10 | 媒体最小暴露 | 文生图草稿位于私有目录，只有确认入库后的 Asset 才进入公开媒体生命周期；视觉标签只发送单张归一化画面 |

## 9. 成功指标与发布门槛

此 Demo 不使用虚构 DAU/留存指标。成功以可验证的任务完成率衡量：

- 一条新文本能从创建到 ready，且每个分段有至少 3 个唯一候选。
- 人工替换后强制刷新，选择与文本修改仍存在。
- 相同幂等键的两次创建返回相同资源。
- `job_fail` 能稳定进入 failed，重试后成功；`ai_degrade` 成功且留下降级记录。
- 有完整素材选择的项目能生成并播放真实 MP4 组合预览；重复请求可复用同一输入指纹的活动任务或有效结果。
- 新素材能完成视觉标签或明确降级，并在运行记录中看到实际 Provider、Model、来源与降级状态。
- 一次文生图任务能从持久化排队走到草稿审核；确认后只创建一个 Asset，并可安全应用到当前片段或仅入库。
- API/Worker 重启后已创建数据不丢，排队/过期任务可继续处理。
- 后端测试、前端 lint/build、验收脚本和公网 smoke 都有真实执行记录。

## 10. 风险与依赖

- 本地 ASR 不需要 Key，但会占用单机 CPU/内存：公网使用 `faster-whisper small/int8` 与单 Worker；无任何可用 Provider 时媒体任务如实失败。
- SQLite 与本地文件：公网必须使用单实例持久卷，禁止部署到临时文件系统。
- 中文检索：通过字符 n-gram 降低分词依赖，但不能完全替代神经语义模型。
- 公网滥用：部署层需要访问口令、大小限制与资源配额，不对无认证高风险上传开放。
- 外部视觉和图像生成依赖第三方兼容接口，其可用性、计费、内容审核和数据保留不由 FrameFlow 保证；未配置时自动标签会降级，文生图会明确不可用而不影响字幕主闭环。

已知问题、影响与后续方向见 `KNOWN_ISSUES.md`，具体验收步骤见 `TEST_PLAN.md`。
