# FrameFlow AI UI 规范

## 1. 体验原则

1. **状态是产品内容**：排队、执行、降级、失败、重试和已保存都要显式呈现。
2. **证据优先于神秘分数**：推荐卡先展示命中词和理由，分数只作补充。
3. **人工决策高于自动结果**：已手选的素材要有明确标记，重新匹配不得静默覆盖。
4. **失败可操作**：错误组件同时回答“发生了什么”、“结果是否保存”和“下一步是什么”。
5. **不用视觉假进度**：处理页只显示服务端已持久化的阶段和百分比。

## 2. 信息架构

主导航固定为：

| 导航 | 路由 | 用户任务 |
| --- | --- | --- |
| 项目 | `/` 或 `/projects` | 查看指标、最近项目、新建项目 |
| 素材库 | `/assets` | 搜索、筛选、上传和编辑素材 |
| 运行记录 | `/runs` | 查看 Provider、策略版本、状态和降级记录 |
| 演示工具 | `/demo` | 设置一次性故障，完成可重复验收 |

项目子路由：

```text
/projects/new
/projects/:projectId/processing?job=:jobId
/projects/:projectId
```

## 3. 全局布局

### 桌面端（≥1280 px）

- 列表和管理页面使用约 216 px 的深色固定侧栏，主内容置于暖白画布中，主体最大宽度约 1260 px。
- 侧栏将“项目 / 素材库 / 运行记录”归入工作空间，“演示工具”作为系统辅助入口；新建项目按钮保持高可见。
- 项目工作台进入专用全屏编辑布局并隐藏常规侧栏，采用三栏：左侧片段列表（约 22%）、中间 16:9 预览/编辑（自适应）、右侧候选/搜索（约 28%）。
- 滚动区独立：片段列表和候选列表可滚动，当前片段的预览与保存状态保持可见。

### 平板（768～1279 px）

- 工作台改为左侧片段 + 右侧预览/候选的两栏。
- 候选与全库搜索在右侧使用标签页切换。

### 手机（<768 px）

- 主导航收起为底部标签或菜单。
- 工作台分为“字幕 / 编辑 / 候选”三个页签，当前片段上下文不丢失。
- 不依赖精确鼠标拖放；提供“上移/下移”后备操作。

## 4. 视觉系统

### 色彩

| Token | 用途 |
| --- | --- |
| `--canvas` / 暖灰白 | 主内容背景，降低长时编辑疲劳 |
| `--surface` / 白 | 卡片、表单、表格 |
| `--sidebar` / 炭黑 | 桌面主导航与品牌区 |
| `--ink` / 近黑 | 主文字，确保对比度 |
| `--accent` / 朱橙 | 主操作、当前选中、焦点和关键进度 |
| `--accent-soft` / 淡橙 | 图标底色、轻量选中和辅助提示 |
| `--success` / 墨绿 | 已完成、已保存 |
| `--warning` / 琥珀 | 降级、补位候选、部分能力 |
| `--danger` / 红 | 任务失败、破坏性操作 |
| 暗色 16:9 画布 | 预览区，与暖白编辑外壳形成信息层级 |

状态不能只用颜色表达，必须同时有文字和/或图标。

整体采用克制的编辑工作室风格：小圆角、细边框、低阴影和清晰分隔线。避免重新引入紫蓝渐变、发光卡片或大面积高饱和背景。

### 字体与密度

- 优先系统中文无衬线字体，不因 Web Font 失效破坏布局。
- 主标题 28～36 px，页标题 22～28 px，正文 14～16 px，辅助信息不小于 12 px。
- 编辑页控制单行密度，不在同一卡片同时展开所有详情。

## 5. 页面规范

### 5.1 项目

**首要信息**：全部项目、素材、处理中任务、待处理任务和最近项目；当前“待处理”指标对应失败任务数量，文档和验收不得将其解释为普通排队任务。

**主操作**：“新建项目”始终在首屏可见。

**最近项目表格**：项目名与短 ID、输入类型、状态、更新时间和打开/更多操作。处理中点击进处理页，就绪点击进工作台，失败点击进错误详情。

**状态**：

- 加载：指标骨架 + 3 张项目卡骨架。
- 空：一句价值说明 + 新建按钮，不显示伪造数据。
- 失败：页内错误条 + 重试加载。

### 5.2 新建项目

顶部用 Tab 切换“粘贴文本”与“上传音视频”。

**文本模式**：项目名、大文本框、字符数、示例文本填充、创建按钮。

**上传模式**：项目名、拖放/文件选择区、支持类型与上限提示、文件摘要。同时明确说明当前 ASR 是否配置；未配置时不显示“一定可转写”。

提交期间禁用两个入口和主按钮，文案改为“正在创建…”，防止重复点击。

### 5.3 处理页

**必须区域**：项目名、总进度、当前阶段、事件时间线、任务 ID/请求 ID（可复制）。

**成功**：进度 100%，自动跳转工作台前显示明确完成状态；提供“立即进入”按钮。

**失败**：错误码、人类可读消息、失败阶段、是否可重试、重试按钮。重试期间按钮禁用，不重复发送。

**取消**：仅 queued/running 可取消；确认对话告知已持久化输入是否保留。

### 5.4 字幕工作台

#### 左栏：原文与片段

- 原始字幕折叠区，默认显示前数行与“展开全文”。
- 片段卡包含顺序、摘要、主题/关键词、选中素材缩略图和手选标记。
- 当前片段有明确边框和 `aria-current`。
- 拖动后先做乐观排序，显示“正在保存顺序”；失败时回滚并 toast。

#### 中栏：预览与编辑

- 顶部 16:9 暗色预览，显示当前选中素材；无选择时显示可操作空状态。
- 组合时间线提供“调整节奏”面板：15/30/60 秒快捷目标、自定义总时长、按字幕字数/当前比例/平均分配，以及选中片段的精确秒数、约 ±0.5 秒和恢复自动。
- 时长输入以后端返回的 40ms 整帧结果回显；自动/手动来源必须可见。批量或单段请求进行时，冻结正文、排序、选材、重匹配和预览创建，避免并发覆盖。
- 字幕、顺序、素材选择或时长改变后隐藏旧视频并显示“原预览已过期”和最新总时长；保存节奏不会自动创建渲染任务，用户明确点击后才重新生成。
- 文本、主题和关键词可编辑。表单明确显示“未保存 / 正在保存 / 已保存 / 保存失败”。
- 版本冲突不能被当成普通失败：告知用户数据已在其他会话更新，允许重新加载。
- “重新匹配”是明确操作，旁边说明不会覆盖人工选择。

#### 右栏：候选与全库搜索

- 默认展示至少 3 张候选卡。
- 每张卡包含预览、名称、标签、命中词、一句中文理由、综合分和“采用”。
- 补位候选显示黄色“低相关补位”，不使用“AI 强烈推荐”等误导文案。
- 已采用卡显示“当前使用”；重复点击不重复写入。
- 全库搜索支持关键词和类型/标签筛选，搜索结果的“替换”与候选卡复用同一保存路径。

### 5.5 素材库

- 顶部搜索、图片/视频类型筛选、标签筛选和“上传素材”。
- 网格卡包含预览、类型、名称、标签/关键词和是否为预置素材。
- 上传表单在选择文件后才启用提交；标签/关键词使用逗号或可删除 chip，不伪装自动标签。
- 无搜索结果时显示“清除筛选”，不显示全局空库引导。

### 5.6 运行记录

表格/列表字段：时间、项目、操作、Provider、Model/策略、状态、耗时、降级标记。

- `rules` 必须原样显示，不替换为更强的模型名。
- 降级运行用警告标记，但如果任务完整成功，状态仍为成功而非失败。
- 详情显示输入摘要/Hash、策略版本和结果摘要，不显示密钥或请求 Header。

### 5.7 演示工具

- 页面标题可使用“演示实验室”，但侧栏导航统一显示“演示工具”。页首警告：“仅用于可重复故障演练，不表示真实外部 Provider 已调用。”
- 三个模式：下一次 AI 降级、下一次任务失败、清除故障。
- 设置后显示“将被下一个匹配任务消费”；消费后恢复 `none`。
- 故障开关不出现在普通编辑页主操作区。

## 6. 全局状态组件

| 状态 | UI 规范 |
| --- | --- |
| Loading | 使用与最终布局形状一致的骨架；按钮使用内联 spinner |
| Empty | 说明为何为空 + 一个有效下一步；搜索空与全局空文案不同 |
| Error | 错误标题、人类可读原因、request_id（可复制）、重试/返回动作 |
| Saving | 局部状态不遮挡整页；防止同一资源重复提交 |
| Saved | 就近显示绿色勾与“已保存”，不只依赖短暂 toast |
| Degraded | 黄色页内条 + 运行记录，说明使用的降级策略 |
| Conflict | 409 专用文案，提供重新加载；不自动覆盖远端版本 |

## 7. 文案规范

推荐：

- “已命中：数据、安全”
- “与素材标签‘云服务、风险’有较高词项重合”
- “相关候选不足，该素材为多样性补位”
- “语音识别服务未配置；已保留上传文件，配置后可重试”

禁止：

- “AI 认为这是最佳选择”（无证据、过度拟人）。
- “转写完成”（在无 ASR 真实结果时）。
- “未知错误”（不带 request_id 和下一步）。
- “操作失败”（未说明数据是否已保存）。

## 8. 可访问性与键盘

- 所有可点击非原生元素必须可聚焦、Enter/Space 触发，优先使用原生 `button/a/input`。
- 聚焦环在浅色和暗色区域都可见，不以移除 outline 代替设计。
- 表单 label 与控件显式关联，校验错误使用 `aria-describedby`。
- Toast 使用合理的 `aria-live`；持久性失败同时放入页内，不只用短暂 toast。
- 图片 `alt` 描述素材名与类型，装饰图片使用空 `alt`。
- 拖动排序有键盘/按钮后备方案，不将拖放作为唯一入口。

## 9. 验收分辨率

交付前至少在以下视口进行手工检查：

- 1440 × 900：三栏工作台无遮挡。
- 1280 × 720：演示录屏视口，主操作不落到首屏外。
- 768 × 1024：平板切换不丢失当前片段。
- 390 × 844：手机端无水平滚动，创建、重试、保存和替换均可操作。

完整交互验收清单见 `TEST_PLAN.md`。
